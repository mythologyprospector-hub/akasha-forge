# Akasha Repository Classes

This document defines the standard classes of repositories within the Akasha ecosystem.

Repository classes allow the ecosystem to organize itself and prevent uncontrolled structural drift.

Every repository in Akasha must belong to one class.

---

# Core System Repositories

These repositories define and stabilize the ecosystem itself.

### foundation

Defines governing principles of the system.

Examples:

akasha-axioms

---

### registry

Tracks ecosystem state and canonical knowledge.

Examples:

akasha-world  
akasha-constellation

---

# Operational Engines

These repositories perform system work.

### discovery

Engines that explore unknowns and generate hypotheses.

Examples:

akasha-discovery  
Great Discovery style systems

---

### forge

Systems responsible for constructing tools and repositories.

Examples:

akasha-forge  
ProjectForge

---

# Knowledge Systems

Repositories that organize and structure domains of knowledge.

### ontology

Structured representations of domain knowledge.

Examples:

phase ontology engines  
knowledge graph systems

---

### interpretation

Systems that translate knowledge into human meaning.

Examples:

mythology engines  
explanation systems  
story-driven interpretation tools

---

# Infrastructure

Systems that support ecosystem operation.

### node

Distribution or runtime nodes.

Examples:

herald nodes  
execution nodes  
data relay nodes

---

### interface

Human interaction layers.

Examples:

dashboards  
visualization tools  
experiment consoles

---

# Experimental Systems

These repositories contain prototypes and research experiments.

### experiment

Exploratory systems not yet stabilized.

Examples:

weird physics experiments  
simulation prototypes  
concept validation tools

---

# Legacy Systems

Projects created before canonical structure existed.

### archive

Historical repositories preserved for reference.

Examples:

older Akasha prototypes  
early tool experiments

---

# Philosophy

Akasha grows through experimentation, but structure prevents chaos.

Classification allows the ecosystem to evolve while remaining understandable.

Every repository has a role.

Every role has a place.
